Нужно доработать CRM и публичную часть по двум направлениям: отзывы и карточка клиента.

1. Отзывы
Сейчас в проекте есть пункт "Отзывы", и в наполненной БД отзывы тоже предусмотрены, но нужен полноценный интерфейс управления отзывами в админке.

Что нужно сделать:
- в админке в разделе "Контент" добавить полноценный подраздел "Отзывы"
- в админке в разделе "Шаблоны" добавить форму/шаблон для добавления и редактирования отзывов
- должна быть возможность:
  - создать отзыв вручную
  - редактировать отзыв
  - удалять отзыв
  - публиковать / скрывать отзыв
  - управлять порядком вывода
- поля у отзыва должны быть нормальные и удобные, минимум:
  - имя клиента
  - фамилия клиента, если есть
  - город / локация, если нужно
  - текст отзыва
  - рейтинг
  - дата
  - источник / тип (ручной, с сайта, после тура, после аренды, после покупки)
  - фото, если поддерживается
  - признак публикации
- если отзыв привязан к сущности, должна быть возможность привязать его к:
  - товару
  - туру
  - аренде
  - заказу
  - клиенту
- на публичной части отзывы должны корректно отображаться без сырых HTML-тегов
- проверить, чтобы сиды/демо-данные по отзывам тоже были совместимы с новой админкой

2. Карточка клиента
Нужно сильно улучшить карточку клиента в CRM.

Сейчас нужно, чтобы в карточке клиента отображалась его история по всем направлениям.

Добавить в карточку клиента отдельные блоки / вкладки / секции:
- аренды клиента
- покупки клиента
- туры клиента
- бронирования / заказы, если это отдельные сущности
- обращения / заявки, если они связаны с клиентом
- сообщения / чаты, если это уже реализовано

Для каждого блока нужно показывать понятную таблицу или список:
- номер
- дата
- статус
- сумма
- состав
- связанный товар / тур / услуга
- ссылка на сущность
- краткая информация

Что важно:
- клиент в CRM должен восприниматься как единая карточка со всей историей
- в карточке должно быть видно:
  - что он арендовал
  - что он покупал
  - в каких турах участвовал
  - какие у него были заказы
- если у клиента нет данных в одном из разделов, показывать пустое состояние, а не ломать UI

3. Технически проверить и исправить
Нужно проверить:
- связи клиента с арендами
- связи клиента с продажами
- связи клиента с турами
- как клиент создается при оформлении заказа
- почему раньше терялись имя и фамилия клиента
- чтобы карточка клиента наполнялась реальными связанными данными, а не пустыми значениями
