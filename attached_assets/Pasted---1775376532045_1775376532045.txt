Исправь баг с доставкой. Сейчас доставка выключена в настройках, но при оформлении заказа все равно продолжает работать. Это неправильная бизнес-логика: checkout и публичный сайт не используют актуальные settings.

Нужно найти root cause и исправить это системно.

==================================================
1. ПРОБЛЕМА
==================================================

Сейчас:
- в Settings доставка выключена
- но в checkout доставка все равно доступна
- пользователь все равно может оформить заказ с доставкой

Так быть не должно.

Если доставка выключена, то:
- на сайте не должна предлагаться доставка
- в checkout не должен отображаться вариант доставки
- backend не должен принимать delivery как валидную опцию
- order create/edit flow должен учитывать settings

==================================================
2. ЧТО НУЖНО ПРОВЕРИТЬ
==================================================

Проверь всю цепочку:

1. Settings save
- действительно ли `delivery.enabled` сохраняется в БД
- действительно ли settings API отдает актуальное значение

2. Public settings API
- получает ли frontend актуальные settings
- не берет ли frontend старые mock/fallback значения

3. Checkout frontend
- скрывает ли UI доставку, если `delivery.enabled = false`
- не кэшируется ли старое состояние
- не остается ли выбранный delivery mode в local state

4. Backend validation
- если фронтенд все же отправил `delivery`, backend должен отклонить это, когда доставка выключена
- нельзя полагаться только на скрытие кнопки на frontend

5. Order create/edit
- deliveryType не должен сохраняться как `delivery`, если доставка выключена
- для admin create/edit тоже проверить логику

==================================================
3. ПРАВИЛЬНАЯ ЛОГИКА
==================================================

Нужно сделать так:

Если:
- `delivery.enabled = false`
и
- `delivery.self_pickup_enabled = true`

То:
- в checkout доступен только самовывоз
- UI доставки скрыт
- backend принимает только pickup

Если:
- `delivery.enabled = false`
и
- `delivery.self_pickup_enabled = false`

То:
- checkout должен показать понятную ошибку / предупреждение в CRM
- публичный checkout не должен предлагать невалидные варианты
- нельзя оформить заказ, пока не включен хотя бы один способ получения

Если:
- `delivery.enabled = true`
и
- `delivery.self_pickup_enabled = true`

То:
- доступны оба варианта
