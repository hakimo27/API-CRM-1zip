==================================================
23. НОРМАЛЬНАЯ ЗАГРУЗКА ФАЙЛОВ, ФОТО И ВЛОЖЕНИЙ
==================================================

Сейчас в системе нельзя ограничиваться только полями с URL-ссылками на картинки. Нужно сделать полноценную загрузку файлов.

Требование:
- пользователь и менеджер должны иметь возможность загружать файлы и изображения через форму
- система должна сохранять файлы локально или через storage driver
- в БД должен сохраняться путь к файлу, а не только внешняя ссылка
- при необходимости можно оставить поддержку external URL, но это должен быть дополнительный режим, а не единственный

### Что нужно поддержать
1. Фото товаров аренды
2. Фото товаров продажи
3. Фото туров
4. Фото статей / страниц / FAQ при необходимости
5. Логотипы / favicon / branding assets
6. Фото филиалов
7. Вложения в чатах, если модуль чатов это уже поддерживает или если архитектура готова
8. Документы / файлы для CRM, если это понадобится дальше

### Что должно быть в UI
Для сущностей с изображениями:
- upload file
- drag and drop upload
- preview
- delete file
- reorder files
- mark as cover/main image
- replace image
- support external URL как дополнительная опция
- alt text / caption если применимо

### Что должно быть в backend
- upload endpoint
- validation по размеру и mime-type
- storage layer
- file naming strategy
- image metadata
- delete endpoint
- reorder endpoint
- cleanup orphan files
- безопасная раздача файлов, если требуется
- support public uploads for website assets

### Что нужно проверить
- products
- sale_products
- tours
- articles
- pages
- settings/branding
- branches
- chat attachments, если уже используется

### Важно
Сейчас нельзя заставлять менеджера вручную вставлять только URL на изображения.
Основной сценарий должен быть:
1. загрузить файл
2. файл сохраняется
3. путь пишется в БД
4. сайт показывает изображение
