4. ИСПРАВИТЬ БАГ С ИНПУТАМИ: ФОКУС СЛЕТАЕТ ПОСЛЕ КАЖДОГО СИМВОЛА
==================================================

Сейчас в форме при вводе текста происходит баг:
- вводишь один символ
- input теряет фокус
- приходится снова кликать
- и так по одному символу

Это критический UX-баг.

Нужно найти root cause и исправить.

Проверь:
1. controlled inputs
2. key prop на form fields
3. re-render всей формы на каждый символ
4. remount компонентов
5. unstable object/array props
6. form state management
7. useEffect/useMemo/useCallback dependencies
8. tab/form sections that remount on state change
9. не пересоздается ли input из-за dynamic key
10. не идет ли reset формы после каждого onChange
11. не ломается ли это из-за autosave logic
12. не вызывается ли save/request/revalidate на каждый символ
13. нет ли проблемы в custom input wrapper

Нужно:
- исправить так, чтобы input не терял focus
- ввод в любых формах был нормальным
- особенно проверить settings, products, categories, customers, branches, pickup points, checkout fields
- убрать любые autosave/remount, из-за которых input пересоздается

Покажи:
- точную причину потери фокуса
- какой компонент/паттерн был неправильным
- как исправил
- где протестировал
