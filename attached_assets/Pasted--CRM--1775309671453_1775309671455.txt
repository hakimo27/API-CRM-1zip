Приведи CRM и публичную систему к логичной, рабочей структуре. Сейчас сущности смешаны, некоторые разделы нелогичны, часть бизнес-процессов работает только частично.

Нужно не просто чинить отдельные баги, а привести систему к понятной архитектуре для менеджера, администратора и склада.

==================================================
1. ГЛАВНАЯ ПРОБЛЕМА: CRM НЕЛОГИЧНО ОРГАНИЗОВАНА
==================================================

Сейчас в CRM смешаны:
- товары
- заказы
- продажа
- аренда
- туры
- брони
- филиалы
- настройки

Из-за этого интерфейс выглядит нелогично:
- заходишь в “Товары продажи”, а там ожидаешь заказы или наоборот
- невозможно быстро понять, где:
  - выкладывать товары
  - смотреть поступившие заказы
  - обрабатывать аренду
  - обрабатывать продажу
  - вести туры
  - смотреть бронирования

Нужно разделить CRM по бизнес-логике.

==================================================
2. НУЖНО ПЕРЕСТРОИТЬ CRM ПО ПОНЯТНЫМ РАЗДЕЛАМ
==================================================

Сделай структуру CRM логичной и явной.

Нужны отдельные разделы:

### Операции
- Dashboard
- Rental Orders / Заказы аренды
- Sale Orders / Заказы продажи
- Tour Bookings / Бронирования туров
- Chats / Чаты
- Notifications / Уведомления
- Logs / Логи

### Каталог
- Rental Products / Товары аренды
- Sale Products / Товары продажи
- Categories / Категории
- Media / Медиафайлы
- Pricing / Тарифы
- Inventory / Инвентарь

### Туры
- Tours / Туры
- Tour Dates / Даты туров
- Rivers / Реки
- Guides/Content if already present

### Клиенты и точки
- Customers / Клиенты
- Branches / Филиалы
- Pickup Points / Пункты самовывоза
  или branches с явной логикой pickup

### Контент
- Pages / Страницы
- Articles / Статьи
- FAQ
- Reviews

### Система
- Users / Пользователи
- Roles / Permissions
- Settings
- Telegram
- Integrations

Нужно, чтобы менеджер сразу понимал:
- здесь мы публикуем товары
- здесь видим заказы
- здесь ведем туры
- здесь бронирования
- здесь клиенты
- здесь пункты выдачи

==================================================
3. НУЖНО УМЕТЬ СОЗДАВАТЬ ЗАКАЗЫ АРЕНДЫ ВРУЧНУЮ ИЗ CRM
==================================================

Сейчас заказы аренды приходят не только с сайта. Менеджер должен иметь возможность создавать rental orders вручную из CRM.

Это обязательная функция.

Нужно:
- кнопка “Создать заказ аренды”
- полноценная форма create rental order
- выбор или создание клиента
- выбор дат
- добавление товаров
- выбор количества
- расчет цены
- выбор:
  - самовывоз
  - доставка
- выбор точки самовывоза
- комментарии менеджера
- комментарии клиента
- статус заказа
- проверка доступности
- резерв товара
- сохранение заказа
- дальнейшее открытие заказа из CRM без ошибки “не найден”

Сценарии:
1. заказ пришел по телефону
2. заказ пришел в мессенджере
3. заказ пришел через Telegram
4. менеджер сам оформляет заказ в CRM
5. заказ оформлен за клиента вручную

Нужно, чтобы ручное создание rental order было таким же полноценным, как заказ с сайта.

==================================================
4. НУЖНО УМЕТЬ СОЗДАВАТЬ ЗАКАЗЫ ПРОДАЖИ ВРУЧНУЮ ИЗ CRM
==================================================

Продажа тоже идет не только через сайт.

Нужно:
- кнопка “Создать заказ продажи”
- выбор клиента
- добавление sale products
- количество
- цена
- доставка / самовывоз
- адрес доставки
- комментарии
- статус заказа
- оплата / payment status
- возможность открыть заказ потом из CRM

Если sale order создается вручную, он должен вести себя так же, как заказ, поступивший с сайта.

==================================================
5. ЧЕТКО РАЗДЕЛИТЬ “ТОВАРЫ” И “ЗАКАЗЫ”
==================================================

Сейчас нужно исправить UX/IA так, чтобы в CRM не было путаницы.

### Товары продажи
Раздел “Товары продажи” должен означать:
- публикация и редактирование товаров для продажи
- фотографии
- характеристики
- цены
- наличие
- SEO
- активность
- категории

Там НЕ должны лежать сами заказы.

### Заказы продажи
Отдельный раздел:
- список заказов продажи
- новые
- в работе
- оплаченные
- отправленные
- завершенные
- отмененные

### Товары аренды
Раздел должен означать:
- каталог rental products
- фото
- характеристики
- тарифы
- залог
- активность
- категории
- availability settings

### Заказы аренды
Отдельный раздел:
- список rental orders
- новые
- подтвержденные
- оплаченные
- собранные
- выданные
- завершенные
- отмененные
- продленные

Это разделение должно быть визуально и логически понятным.
