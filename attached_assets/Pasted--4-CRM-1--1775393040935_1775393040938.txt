Исправь и доработай 4 важных блока CRM и сайта:

1. Характеристики товаров есть на сайте, но их нельзя нормально редактировать
2. Нужны шаблоны тарифов, чтобы не добавлять тарифы вручную в каждый товар
3. Нужен конструктор/шаблоны характеристик
4. Нужен нормальный текстовый редактор с фото и таблицами

Сделай это системно, а не как временные костыли.

==================================================
1. ХАРАКТЕРИСТИКИ ТОВАРОВ ДОЛЖНЫ НОРМАЛЬНО РЕДАКТИРОВАТЬСЯ
==================================================

Сейчас на сайте характеристики уже отображаются, например:
- 1 мест
- 0.9 кг вес

Но в CRM их нельзя нормально редактировать. Это нужно исправить.

Нужно сделать полноценный редактор характеристик для:
- rental products
- sale products
- при необходимости туров, если там тоже есть structured specs

Сейчас нельзя оставлять характеристики как:
- захардкоженные значения
- неудобный raw json
- неуправляемый блок

Нужен нормальный UI.

==================================================
2. НУЖЕН КОНСТРУКТОР ХАРАКТЕРИСТИК
==================================================

Сделай в форме товара удобный конструктор характеристик.

Менеджер должен мочь:
- добавить характеристику
- выбрать тип характеристики
- задать название
- задать значение
- задать единицу измерения
- менять порядок
- удалять характеристику
- редактировать существующую характеристику

Нужно поддержать формат вида:
- Мест: 1
- Вес: 0.9 кг
- Длина: 320 см
- Грузоподъемность: 150 кг
- Материал: ПВХ
- Комплектация: ...

То есть структура должна быть:
- label / название
- value / значение
- unit / единица измерения
- sortOrder
- optional key / slug, если нужен внутренне

Важно:
- менеджер не должен руками писать JSON
- характеристики должны быть удобными и понятными

==================================================
3. ХАРАКТЕРИСТИКИ ДОЛЖНЫ ОТОБРАЖАТЬСЯ НА САЙТЕ ИЗ CRM
==================================================

Нужно проверить и исправить:
- если характеристика добавлена/изменена в CRM, она должна реально обновляться на сайте
- порядок характеристик должен соблюдаться
- единицы измерения должны отображаться корректно
- не должно быть кривых склеек вроде “0.9 кг вес”, если формат можно сделать лучше

Нормальный вывод на сайте:
- Мест: 1
- Вес: 0.9 кг

А не хаотичный текст.

Если сейчас frontend собирает это неправильно:
- исправь rendering layer

==================================================
4. НУЖНЫ ШАБЛОНЫ ХАРАКТЕРИСТИК
==================================================

Чтобы не собирать характеристики вручную для каждого товара с нуля, нужны шаблоны характеристик.

Сделай отдельную сущность или логику:
- шаблоны характеристик
или
- presets/spec templates

Примеры:
- Шаблон для байдарки
- Шаблон для SUP
- Шаблон для весла
- Шаблон для жилета
- Шаблон для sale product определенного типа

Шаблон должен содержать набор полей:
- Мест
- Вес
- Длина
- Ширина
- Грузоподъемность
- Материал
- Комплектация
и т.д.

Менеджер должен мочь:
- выбрать шаблон для товара
- автоматически получить заготовку характеристик
- потом отредактировать значения

Важно:
- шаблон не должен навечно блокировать редактирование
- он должен помогать заполнять быстрее

==================================================
5. НУЖНЫ ШАБЛОНЫ ТАРИФОВ
==================================================

Сейчас тарифы не должны добавляться вручную в каждый товар заново.

Нужно сделать шаблоны тарифов.

Например:
- Базовый шаблон аренды

С тарифами:
1. Будни
2. Выходные
3. Неделя

Менеджер должен мочь:
- создать шаблон тарифов
- отредактировать шаблон тарифов
- выбрать шаблон для товара
- применить шаблон к товару
- затем при необходимости подправить тарифы в конкретном товаре

Нужно:
- отдельный раздел:
  - Настройки → Тарифы
  или
  - Тарифные шаблоны
- support multiple templates
- support apply template to product

Важно:
- это должно избавить от ручного создания тарифов для каждого товара

==================================================
6. ШАБЛОНЫ ТАРИФОВ И ХАРАКТЕРИСТИК ДОЛЖНЫ БЫТЬ ЛОГИЧНЫ В CRM
==================================================

Сделай понятную структуру:

В CRM должны быть:
- Характеристики / шаблоны характеристик
- Тарифы / шаблоны тарифов

И в карточке товара:
- выбрать шаблон характеристик
- выбрать шаблон тарифов
- применить шаблон
- потом редактировать итоговый набор

Важно:
- менеджер должен понимать, откуда данные взялись
- и при этом не быть ограничен шаблоном навсегда

==================================================
7. НУЖЕН НОРМАЛЬНЫЙ ТЕКСТОВЫЙ РЕДАКТОР
==================================================

Сейчас для описаний, страниц, статей, SEO-текстов и длинных блоков нужен полноценный редактор, а не примитивный textarea.

Нужно внедрить нормальный rich text editor для:
- страниц
- статей
- описаний товаров
- описаний туров
- SEO/контентных блоков, где длинный текст
- возможно FAQ answers, если уместно

Редактор должен поддерживать:
- заголовки
- абзацы
- жирный/курсив
- списки
- ссылки
- вставку изображений
- вставку таблиц
- выравнивание / базовое форматирование
- нормальную работу с HTML/structured content
- сохранение без поломки разметки

- работать с несколькими абзацами и блоками

То есть менеджер должен мочь оформить страницу “О нас”, описание тура или описание товара нормально.
