Доработай deployment package и инструкции. Нужно сделать нормальный автоустановщик одной командой, переписать инструкции под него и убрать из проекта/документации все лишние старые упоминания, из-за которых сейчас путаница.

==================================================
1. НУЖНА ОДНА ПОНЯТНАЯ КОМАНДА АВТОУСТАНОВКИ
==================================================

Сейчас deployment должен быть приведен к виду, где установка запускается одной понятной командой.

Нужно сделать основной сценарий таким:

sh deploy/install.sh

Именно эта команда должна:
- проверить зависимости
- проверить `.env`
- поднять postgres
- поднять redis
- поднять backend
- поднять web
- поднять admin
- поднять nginx
- дождаться готовности БД
- применить миграции
- выполнить bootstrap/seed, если включено в env
- создать superadmin, если включено в env
- показать финальный статус

Если нужны дополнительные сценарии, оставить:
- `sh deploy/update.sh`
- `sh deploy/backup-db.sh`
- `sh deploy/restore-db.sh`

Но главный сценарий должен быть один и понятный.

==================================================
2. ПЕРЕПИСАТЬ ИНСТРУКЦИИ ПОД АВТОУСТАНОВЩИК
==================================================

Нужно обновить:
- `README.md`
- `README_DEPLOY.md`

Так, чтобы инструкции больше не были разрозненными и ручными.

Нужен понятный flow:

1. скопировать проект
2. сделать:
   cp .env.example .env
   nano .env
3. запустить:
   sh deploy/install.sh

И все.

Нужно убрать из документации хаотичные старые шаги, дубли, временные инструкции и все, что противоречит новому автоустановщику.

==================================================
3. ДОБАВИТЬ FIRST INSTALL ЧЕРЕЗ ENV
==================================================

Нужно, чтобы при первой установке через `.env` можно было включать:

- запуск миграций
- запуск базового seed
- запуск demo/test seed
- создание superadmin

Добавить и описать переменные типа:
- APP_FIRST_INSTALL=true|false
- APP_RUN_MIGRATIONS=true|false
- APP_RUN_SEED=true|false
- APP_RUN_DEMO_SEED=true|false
- BOOTSTRAP_SUPERADMIN_CREATE=true|false
- BOOTSTRAP_SUPERADMIN_EMAIL=
- BOOTSTRAP_SUPERADMIN_PASSWORD=
- BOOTSTRAP_SUPERADMIN_FIRST_NAME=
- BOOTSTRAP_SUPERADMIN_LAST_NAME=
- BOOTSTRAP_SUPERADMIN_PHONE=

И install script должен реально это использовать.
