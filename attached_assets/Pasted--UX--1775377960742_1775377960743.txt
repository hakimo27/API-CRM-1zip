Исправь UX и бизнес-логику аренды: сейчас ошибки по доступности, датам и остаткам появляются слишком поздно — уже на этапе оформления заказа. Это неправильно. Проверка должна происходить раньше, до добавления в корзину, и пользователь должен заранее видеть, что доступно, а что нет.

Также нужно добавить визуальные счетчики уведомлений по разделам CRM.

==================================================
1. ОШИБКИ ПО АРЕНДЕ НЕ ДОЛЖНЫ ВОЗНИКАТЬ ТОЛЬКО НА ЭТАПЕ ОФОРМЛЕНИЯ
==================================================

Сейчас пользователь/менеджер доходит до checkout, и только там видит:
- товар недоступен
- пересечение дат
- недостаточно единиц
- товар уже забронирован
- нет доступности на выбранные даты

Это плохой UX.

Нужно перенести проверку availability раньше:

1. на карточку товара
2. в каталог
3. в форму выбора дат
4. перед добавлением в корзину
5. при изменении количества
6. при изменении филиала/самовывоза, если это влияет

Правильная логика:
- пользователь выбирает даты
- система сразу показывает доступность
- если товар недоступен, нельзя добавить его в корзину
- если доступно частично, ограничивать количество
- если есть конфликт дат, показывать это сразу
- checkout не должен быть первой точкой, где вскрывается проблема

==================================================
2. ДОБАВИТЬ ПРОВЕРКУ ДОСТУПНОСТИ ДО ДОБАВЛЕНИЯ В КОРЗИНУ
==================================================

Нужно сделать pre-check availability flow.

Сценарий:
1. пользователь открыл карточку товара
2. выбрал даты аренды
3. выбрал количество
4. нажал “Проверить доступность” или доступность проверяется автоматически
5. система показывает:
   - доступно / недоступно
   - сколько единиц доступно
   - возможные ближайшие даты
   - сообщение, если товар занят

Если товар недоступен:
- кнопка “Добавить в корзину” disabled
- показать понятное сообщение
- по возможности показать ближайшие свободные даты

Если товар доступен:
- показывать подтверждение
- разрешить добавить в корзину

==================================================
3. ПОКАЗЫВАТЬ СВОБОДНЫЕ ДАТЫ / БЛИЖАЙШИЕ ДОСТУПНЫЕ ОКНА
==================================================

Нужно сделать более удобный UX по датам.

Если выбранный период недоступен, система должна не просто писать “нет”, а предлагать варианты.

Нужно:
- показать ближайшие свободные даты
- показать ближайшее доступное окно аренды
- можно сделать:
  - список ближайших окон
  - mini calendar availability
  - подсказку “свободно с 14 мая”
  - подсказку “на эти даты доступно 2 из 5”

Минимально:
- если даты заняты, показать:
  - ближайшую доступную дату начала
  - или ближайший свободный диапазон

Для CRM менеджера:
- можно показывать availability calendar / availability summary

Для сайта:
- достаточно user-friendly availability hint

==================================================
4. НУЖЕН ВИЗУАЛЬНЫЙ STATUS / NOTICE ДО ДОБАВЛЕНИЯ В КОРЗИНУ
==================================================

Сделай понятные визуальные статусы:

- Доступно
- Осталось мало
- Нет на выбранные даты
- Требует уточнения у менеджера
- Доступно частично
- Свободно с другой даты

Нужно:
- badge / info block рядом с датами и количеством
- визуально понятный state
- не только текстовая ошибка внизу формы

==================================================
5. CHECKOUT ВСЕ РАВНО ДОЛЖЕН ПОВТОРНО ПРОВЕРЯТЬ, НО НЕ БЫТЬ ПЕРВЫМ БАРЬЕРОМ
==================================================

Важно:
- checkout все равно должен валидировать availability повторно
- но это должна быть уже финальная страховка, а не первая проверка

То есть:
- pre-check на карточке и до корзины
- re-check в корзине/checkout
- если между этими шагами кто-то занял товар, тогда уже нормальная ошибка на checkout допустима

==================================================
6. ДОРАБОТАТЬ КОРЗИНУ АРЕНДЫ
==================================================

В корзине аренды нужно:
- показывать выбранные даты
- показывать количество
- показывать доступность
- если после добавления что-то изменилось, подсвечивать проблему
- предлагать изменить даты/количество до перехода к оформлению
