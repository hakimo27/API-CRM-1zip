Сделай production-ready проект для аренды байдарок на Laravel с PostgreSQL, Redis, API, CRM и публичным сайтом.

Проект нужен для проката байдарок, SUP и дополнительного оборудования. Нужна не просто витрина, а полноценная система: сайт + CRM + складской учет + бронирование + чат + SEO + API для будущего бота и внешних интеграций.

=== ГЛАВНАЯ ЦЕЛЬ ===

Создай архитектурно правильный Laravel-проект, который можно расширять без переделки ядра.

Нужны:
1. Публичный сайт
2. CRM / админка
3. REST API
4. Система бронирования
5. Складской учет
6. SEO-структура
7. Чат-виджет на сайте
8. Переписка через Telegram-бота
9. Роли и права
10. Подготовка под оплату и дальнейшие интеграции

Используй чистую и расширяемую архитектуру. Не делай монолит из контроллеров с бизнес-логикой. Вынеси логику в сервисы, actions, policies, enums, DTO где уместно.

=== ТАРИФЫ И РАСЧЕТ ЦЕНЫ ===

Нужен отдельный сервис расчета аренды: PriceCalculatorService.

Поддержать тарифы:

Для байдарок / SUP / палаток:
- weekend
- week
- weekday
- may_holidays

Для части дополнительного оборудования:
- weekend
- week

Важная логика:
- четверг после 18:00 в некоторых случаях идет без доплаты к выходным
- будний тариф может иметь минималку 3 дня
- старт аренды в субботу/воскресенье — сложный кейс
- часть кейсов нужно считать как approximate price
- система должна уметь возвращать:
  - exact_price
  - approximate_price
  - manager_review_required = true/false
  - pricing_comment

То есть сервис расчета не должен быть примитивным умножением дней на цену.
