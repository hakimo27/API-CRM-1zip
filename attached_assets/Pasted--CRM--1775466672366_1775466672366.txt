Исправь раздел отзывов в CRM и одновременно доработай полноценную карточку клиента с полной историей.

1. Ошибка создания отзывов
Сейчас создание отзыва в CRM дает 404, потому что фронт использует неверный endpoint.

Подтверждено:
- CRM отправляет `POST /api/content/reviews/admin`
- сервер отвечает `404 Not Found`

При этом API устроен так:
- `GET /api/content/reviews/admin` — список отзывов для админки
- `POST /api/content/reviews` — создание отзыва
- `PATCH /api/content/reviews/:id` — редактирование
- `DELETE /api/content/reviews/:id` — удаление

Что нужно исправить:
- во frontend CRM заменить создание отзыва:
  - было: `POST /api/content/reviews/admin`
  - должно быть: `POST /api/content/reviews`
- проверить все остальные запросы раздела отзывов:
  - список
  - создание
  - редактирование
  - удаление
  - публикация / approve, если есть
- убедиться, что admin list остается на `GET /api/content/reviews/admin`
- проверить форму создания/редактирования отзывов в CRM

2. Полноценная карточка клиента
Сейчас нет нормальной карточки клиента с его историей. Нужно сделать полноценную клиентскую карточку в CRM, где видна вся история клиента по всем сущностям.

Нужно добавить в карточку клиента отдельные блоки / вкладки / секции:
- аренды клиента
- покупки клиента
- туры клиента
- бронирования клиента
- заказы клиента
- заявки клиента, если они есть
- отзывы клиента, если они есть
- чаты клиента
- сообщения клиента / история переписки

То есть в карточке клиента должна быть единая история вообще всего, что связано с этим клиентом.

Для каждого блока нужно показывать понятный список или таблицу:
- номер / ID
- дата
- статус
- сумма
- связанная сущность
- краткое описание
- ссылка на подробности
- если это чат — дата последнего сообщения, статус, количество сообщений, ссылка перейти в диалог

3. Связи клиента
Нужно проверить и исправить связи клиента со всеми сущностями:
- аренды
- покупки
- туры
- бронирования
- заказы
- заявки
- отзывы
- чаты
- сообщения

Важно:
- история должна собираться не фрагментарно, а централизованно по клиенту
- если у клиента есть чат, он должен отображаться в карточке клиента
- если клиент делал заказ, ездил в тур, брал аренду, оставлял отзыв — все это должно быть видно в одной карточке
- если чего-то нет, показывать пустое состояние, а не ломать интерфейс

4. Отображение чатов в карточке клиента
Особенно важно:
- в карточке клиента должен быть блок чатов
- там должны быть:
  - список диалогов клиента
  - статус диалога
  - дата последнего сообщения
  - количество сообщений
  - кнопка/ссылка открыть чат
- если чат связан с конкретным заказом/туром/товаром, это тоже желательно показывать

5. Что проверить технически
- почему раньше терялись имя и фамилия клиента
- как клиент создается при оформлении заказа
- как клиент связывается с арендой
- как клиент связывается с покупкой
- как клиент связывается с туром
- как клиент связывается с чатом
- как клиент связывается с отзывом

6. В ответе показать
1. почему был 404 в отзывах
2. какой endpoint был неправильный
3. какие файлы изменены по отзывам
4. какие файлы изменены по карточке клиента
5. как теперь устроена карточка клиента
6. какие сущности теперь отображаются в истории клиента
7. как реализован блок чатов клиента
8. какие сценарии протестированы:
   - создание/редактирование отзывов
   - открытие карточки клиента
   - история аренд
   - история покупок
   - история туров
   - история заказов
   - история чатов и сообщений
