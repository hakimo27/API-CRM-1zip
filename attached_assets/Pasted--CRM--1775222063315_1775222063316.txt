6. ОБЯЗАТЕЛЬНОЕ РАЗДЕЛЕНИЕ: SHOP / RENTAL / TOUR / INFO
====================

Нужно, чтобы при создании товара или сущности можно было указать, куда он относится.

Добавь поле:
- product_mode / domain_scope

Допустимые значения:
- rental
- sale
- both
- tour_related
- info_related

Логика:
1. rental — товар показывается только в аренде
2. sale — товар показывается только в продаже
3. both — товар можно показывать и в аренде, и в продаже
4. tour_related — товар/услуга для туров
5. info_related — используется только как контентная/связанная сущность

В UI CRM:
- отдельный селектор “Где показывать товар?”
- чекбоксы или multi-select:
  - Показать в аренде
  - Показать в продаже
  - Показать в турах
  - Использовать в инфо-разделе

Важно:
- фильтрация товаров по витрине должна работать корректно
- один и тот же товар может быть только rental, только sale или both
- для both должна быть возможность иметь разные цены:
  - rental tariffs
  - sale price
