Дополнительно реализуй PWA (Progressive Web App) в самом проекте и доведи это до конца так, чтобы после установки на сервере PWA уже была встроена и работала, а не требовала ручной доделки.

Проблема:
- сейчас PWA не работает нормально
- браузер не предлагает установку приложения
- нет нормального install flow
- push-уведомления не работают
- после деплоя на сервер PWA не считается готовой

Что нужно сделать:

1. Полностью внедрить PWA в проект
Проверить и исправить:
- manifest.webmanifest или manifest.json
- service worker
- регистрацию service worker
- иконки приложения
- theme_color
- background_color
- display
- orientation
- start_url
- scope
- offline fallback, если предусмотрен
- корректную работу на production домене

2. Сделать, чтобы браузер реально предлагал установку
Нужно:
- корректно обрабатывать `beforeinstallprompt`
- сделать кнопку установки, если это нужно по UX
- не ломать штатное предложение браузера установить приложение
- проверить desktop browser + Android Chrome
- учесть ограничения iOS, но сделать максимально корректный сценарий и там

3. Встроить PWA в install/deploy flow
Нужно, чтобы после установки проекта на сервер:
- PWA уже была доступна
- manifest отдавался корректно
- service worker был доступен
- иконки были доступны
- браузер видел сайт как installable app
- не требовалась ручная правка после деплоя

То есть install script, build flow и deploy flow должны учитывать PWA сразу.

4. Добавить push-уведомления
Нужно реализовать/починить полноценный web push flow:
- запрос разрешения на уведомления
- создание push subscription
- сохранение subscription на backend
- генерация и использование VAPID keys
- отправка push через backend
- обработка push в service worker
- показ уведомления на устройстве

Push-уведомления должны поддерживать как минимум:
- новые сообщения чата
- новые чаты
- новые заказы
- новые заявки

5. Внести это в install.sh и документацию
Нужно отразить все это в:
- deploy/install.sh
- deploy/update.sh
- README.md
- README_DEPLOY.md
- .env.example

Что должно быть добавлено:
- env-переменные для PWA/push
- env-переменные для VAPID keys
- инструкция по генерации VAPID keys
- описание install flow
- описание, как проверить PWA после деплоя
- описание, как проверить push-уведомления

6. Если нужны переменные окружения, добавить их
Например:
- PWA_APP_NAME
- PWA_SHORT_NAME
- PWA_THEME_COLOR
- PWA_BACKGROUND_COLOR
- WEB_PUSH_PUBLIC_KEY
- WEB_PUSH_PRIVATE_KEY
- WEB_PUSH_SUBJECT
или другие, если у проекта своя структура

7. Учитывать бренд и настройки проекта
PWA должна брать из актуальных настроек:
- название
- short name
- favicon/app icons
- theme colors
- branding

То есть PWA не должна оставаться со старыми тестовыми значениями.

8. Протестировать сценарии
Нужно проверить:
- production build
- manifest доступен
- service worker регистрируется
- сайт предлагает установку
- установленное приложение открывается standalone
- push permission запрашивается
- push subscription сохраняется
- push на новое сообщение реально приходит

Формат ответа:
1. какие файлы изменены для PWA
2. какие файлы изменены для push
3. как теперь работает installable PWA
4. как теперь работает service worker
5. как реализованы push-уведомления
6. какие env-переменные добавлены
7. что добавлено в install.sh
8. что добавлено в update.sh
9. что добавлено в README и README_DEPLOY
10. какие проверки проведены на production/deploy сценарии

Нужно не просто “добавить PWA”, а сделать так, чтобы после установки проекта на сервер она уже была встроена, работала и была частью deploy flow.
