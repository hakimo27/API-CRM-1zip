Доработай настройки чат-виджета и логику обработки обращений. Сейчас в проекте нужно сделать полноценный operational flow для чата: рабочие часы, автоответ, offline-форма, создание карточки клиента и уведомления. Это должно настраиваться из CRM и реально работать на сайте.

==================================================
1. В НАСТРОЙКАХ ВИДЖЕТА НУЖНО УКАЗЫВАТЬ ГРАФИК И ЧАСЫ РАБОТЫ
==================================================

Сейчас в разделе:
- Настройки → Настройки чата
или
- Настройки → Виджет / Чат

нужно добавить полноценный блок графика работы.

Нужно настроить:
- timezone
- рабочие дни
- время начала работы
- время окончания работы
- отдельный график по дням недели
- возможность выключить конкретный день
- праздничный / override режим, если это не слишком сложно
- fallback логика, если график не задан

Минимально должно быть:
- Пн: 10:00–19:00
- Вт: 10:00–19:00
- Ср: 10:00–19:00
- Чт: 10:00–19:00
- Пт: 10:00–19:00
- Сб: 11:00–17:00
- Вс: выходной

Нужно:
- чтобы график реально сохранялся
- чтобы виджет реально использовал эти часы
- чтобы сайт понимал: сейчас рабочее время или нет

==================================================
2. НУЖЕН ТЕКСТ АВТОСООБЩЕНИЯ, КОГДА ЧАТ НЕ РАБОТАЕТ
==================================================

Сейчас должен быть отдельный текст автоответа для нерабочего времени.

Добавь в настройки чата поля:
- offline_message_enabled
- offline_message_text
- after_hours_autoreply_enabled
- after_hours_autoreply_text
- no_manager_available_text

Пример логики:
- если сейчас нерабочее время:
  - показать сообщение:
    “Сейчас мы вне рабочего времени. Оставьте ваши контакты, и мы свяжемся с вами в рабочие часы.”
- если менеджер недоступен:
  - показать сообщение:
    “Сейчас все менеджеры заняты. Оставьте сообщение, и мы ответим вам как можно скорее.”

Важно:
- текст должен редактироваться из CRM
- не быть захардкоженным в коде
- реально использоваться виджетом

==================================================
3. ВНЕ РАБОЧЕГО ВРЕМЕНИ ДОЛЖНА ПОЯВЛЯТЬСЯ ФОРМА ОБРАТНОЙ СВЯЗИ
==================================================

Если чат находится вне рабочего времени, то вместо “мертвого” виджета должна открываться форма обратной связи.

Форма должна содержать:
- имя
- телефон
- email, если включено
- сообщение
- источник страницы / URL
- optional тип обращения

Настройки формы:
- collect_name_enabled
- collect_phone_enabled
- collect_email_enabled
- require_name
- require_phone
- require_email

Нужно:
- если нерабочее время, показывать offline form
- если поля обязательны, не давать отправить пустыми
- после отправки не просто показывать success, а реально создавать данные в системе

==================================================
4. ПРИ OFFLINE FORM ДОЛЖНА СОЗДАВАТЬСЯ КАРТОЧКА КЛИЕНТА
==================================================

Если пользователь оставил данные через offline form, система должна не терять это обращение.

Нужно:
- создать или найти клиента
- если клиент уже есть по телефону/email, не плодить дубль
- если клиента нет — создать нового
- привязать обращение/чат к клиенту
- сохранить:
  - имя
  - телефон
  - email
  - сообщение
  - источник страницы
  - время обращения
  - статус

Нужно завести понятный статус для такого клиента/лида, например:
- новый лид
- требуется связь
- новое обращение
- ожидание связи

Сделай один понятный вариант и используй его системно.

Например:
- customer.status = requires_contact
или
- lead_status = new_request
или
- contact_status = awaiting_callback

Если в текущей схеме такого поля нет:
- предложи минимальное поле
- добавь аккуратно
- не ломай текущую модель customers

==================================================
5. НУЖНО СОЗДАВАТЬ НЕ ТОЛЬКО КЛИЕНТА, НО И ОБРАЩЕНИЕ / ДИАЛОГ
==================================================

Если человек оставил форму вне рабочего времени, это не просто контакт.

Нужно:
- создать запись обращения
- либо как chat_session со special mode
- либо как inbound contact request
- либо как offline dialog
- но чтобы это было видно менеджеру в CRM

Нужно, чтобы менеджер видел:
- от кого обращение
- телефон
- email
- сообщение
- с какой страницы
- когда оставлено
- статус обработки
- связанную карточку клиента

==================================================
6. УВЕДОМЛЕНИЯ ДОЛЖНЫ РАБОТАТЬ НА ВСЕ ЭТИ СОБЫТИЯ
==================================================

Нужно включить уведомления на:

1. новый offline request
2. новый клиент, созданный через offline form
3. новый диалог
4. новое сообщение
5. обращение вне рабочего времени
6. повторное сообщение в существующий диалог

Минимально менеджер должен получать:
- уведомление в CRM
- запись в logs / notification logs
- Telegram уведомление, если интеграция включена

Нужно:
- чтобы по offline form менеджер не узнавал о новом клиенте случайно
- событие должно быть видно сразу

==================================================
7. В SETTINGS ДОЛЖЕН БЫТЬ ПОНЯТНЫЙ РАЗДЕЛ НАСТРОЕК ЧАТА
==================================================

Сделай в CRM полноценный раздел:
- Настройки → Настройки чата

Блоки внутри:
1. Основные
- включить чат
- включить виджет
- показывать на страницах

2. Поведение
- приветственное сообщение
- placeholder
- автоответ в рабочее время
- автоответ в нерабочее время

3. Рабочие часы
- дни недели
- часы работы
- timezone

4. Offline form
- включить форму
- собирать имя
- собирать телефон
- собирать email
- обязательность полей
- текст перед формой
- текст после успешной отправки

5. CRM / lead handling
- создавать клиента автоматически
- статус нового клиента/лида
- создавать диалог автоматически
- включить уведомления менеджерам

Все должно быть:
- по-русски
- без технических английских ключей под полями
- одной формой на вкладку/секцию
- с рабочим сохранением

8. ДОЛЖНА БЫТЬ ЧЕТКАЯ ЛОГИКА: РАБОЧЕЕ ВРЕМЯ / НЕРАБОЧЕЕ ВРЕМЯ
==================================================

Сделай явную логику:

Если рабочее время:
- можно открыть обычный чат
- работает live dialog
- могут работать pre-chat fields, если включены

Если нерабочее время:

- показать offline message
- показать форму обратной связи
- после отправки:
  - создать клиента
  - создать обращение/диалог
  - отправить уведомление менеджеру
