Дополнительно полностью переработай всплывающие уведомления в проекте.

Проблема:
- нужны не только рабочие уведомления, но и нормальный внешний вид
- текущие всплывающие уведомления либо отсутствуют, либо выглядят плохо / слишком стандартно / неинформативно
- нужно сделать их заметными, современными и удобными

Что нужно сделать:

1. Переработать UI всплывающих уведомлений
- изменить стиль toast / popup / in-app notifications
- сделать современный аккуратный дизайн
- убрать слишком стандартный / сырой вид
- привести уведомления к единому стилю проекта

2. Сделать разные типы уведомлений визуально различимыми
Нужно, чтобы по внешнему виду отличались:
- новое сообщение
- новый чат
- новый заказ
- новая заявка
- успешное действие
- ошибка
- предупреждение
- информационное сообщение

3. Улучшить содержимое всплывающих уведомлений
- сделать понятные заголовки
- сделать короткое, но полезное описание
- показывать ключевые данные:
  - имя клиента
  - номер заказа
  - тип события
  - время, если нужно
- убрать технические и сырые тексты

4. Улучшить UX уведомлений
- правильное время показа
- возможность закрыть вручную
- нормальные анимации появления/исчезновения
- стек уведомлений без хаоса
- адаптация под мобильные устройства
- не перекрывать важные элементы интерфейса
- не быть слишком маленькими или слишком громоздкими

5. Добавить переход по уведомлению
- если это уведомление о чате, при нажатии переходить в чат
- если о заказе — открывать заказ
- если о заявке — открывать заявку
- если о сообщении — открывать нужный диалог
- поведение должно быть логичным и удобным

6. Проверить разные контексты
Нужно, чтобы уведомления нормально работали:
- в админке
- в CRM
- в PWA
- на мобильных устройствах
- при открытом сайте
- при открытой админке
- при реальном времени через websocket / polling / push

7. Если есть push-уведомления, привести и их к единому стилю
- заголовки
- текст
- иконки
- payload
- действия по клику
- branding
- чтобы push выглядели согласованно с остальной системой

8. Убрать старый бренд и сырые тексты
- в уведомлениях не должно быть старых названий типа kayakrent / КаякРент
- использовать актуальный бренд и нормальные тексты

Формат ответа:
1. какие типы всплывающих уведомлений переработаны
2. какие файлы изменены
3. как теперь выглядят уведомления
4. как реализованы стили, поведение и переходы
5. как это протестировано на desktop/mobile/PWA
