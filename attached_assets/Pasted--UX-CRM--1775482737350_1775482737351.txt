Что нужно проверить и исправить:

- проверить состояния:
  - focus
  - hover
  - active
  - selected
  - keyboard focus
- особенно проверить:

  - модалки
  - карточки

  - отступы съехали
  - элементы выглядят сломано
  - активное состояние не отличается от обычного
  - на keyboard navigation непонятно, какой элемент выбран

2. FAQ / Вопрос-ответ
- проверить блок FAQ
- сейчас вопрос/ответ визуально отображаются некорректно

  - читаемый вопрос

3. Контент из редактора рендерится с HTML-тегами и в плохой обертке
Нужно проверить все страницы, где контент создается через редактор.

Проблема:
- контент из редактора выводится как HTML внутри слабой/неправильной обертки
- визуально это выглядит сыро
- местами теги рендерятся без нормальной стилизации
- нет единого оформления для rich text контента

Что нужно сделать:
- найти все места, где используется контент из редактора
- сделать нормальную универсальную обертку для rich content / prose content
- автоматически стилизовать HTML, который приходит из редактора:
  - h1-h6
  - p
  - ul / ol / li
  - blockquote
  - strong / em
  - a
  - img
  - table
  - hr
  - code / pre
- чтобы контент выглядел аккуратно и единообразно и на сайте, и в CRM, где это нужно
- убрать ощущение сырого HTML в контейнере

4. OG / SEO / meta разметка
Нужно проверить, что на контентных страницах есть корректная Open Graph разметка.

Проблема:
- где-то отсутствует og-разметка
- страницы с контентом не получают нормальные og:title / og:description / og:image
- при шаринге ссылки предпросмотр может быть пустым или плохим

Что нужно сделать:
- проверить все контентные страницы:
  - статьи
  - страницы
  - FAQ-страницы / информационные страницы
  - товары
  - туры
  - реки / маршруты
- добавить автоматическую генерацию OG-разметки там, где контент создается в редакторе
- автоматически формировать:
  - title
  - description
  - og:title
  - og:description
  - og:image
  - twitter:title
  - twitter:description
  - canonical
- если редакторский контент не содержит ручных SEO-полей:
  - брать заголовок из сущности
  - description генерировать из первого осмысленного текста без HTML
  - image брать из главной картинки страницы / статьи / товара / тура
- если SEO-поля заданы вручную — использовать их с приоритетом

5. Автоматическая очистка HTML для meta
- нельзя вставлять HTML в meta description
- нужно:
  - удалить теги
  - нормализовать пробелы
  - обрезать длину до разумного лимита
  - не ломать кириллицу
- генерация meta/og должна работать автоматически для контентных сущностей

6. Единый rich text renderer
