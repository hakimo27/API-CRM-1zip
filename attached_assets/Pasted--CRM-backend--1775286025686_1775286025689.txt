==================================================

Сейчас нужно не просто “подключить Redis”, а встроить его в архитектуру проекта правильно.

Проверь и реализуй:

1. Redis как обязательный сервис в docker-compose
2. Redis env переменные в .env.example
3. Redis connection в backend
4. Использование Redis для:
   - кэша
   - rate limit
   - очередей / jobs
   - временных chat/session данных при необходимости
   - realtime/polling optimization
   - блокировок на фоновые задачи импорта / Telegram sync / notifications
5. Healthcheck Redis
6. Логи ошибок Redis
7. Graceful fallback, если Redis недоступен

Если сейчас в проекте есть функционал:
- чат
- Telegram sync
- уведомления
- импорт
- очереди
то он должен либо использовать Redis, либо ты должен явно показать, почему нет.

Нужно:
- добавить redis service в docker-compose
- описать Redis в README
- дать env:
  - REDIS_HOST
  - REDIS_PORT
  - REDIS_PASSWORD
  - REDIS_URL
  - CACHE_TTL_DEFAULT
- показать, где Redis реально используется в коде
