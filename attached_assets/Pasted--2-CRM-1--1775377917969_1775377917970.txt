Исправь 2 критические проблемы в CRM и чат-виджете.

Проблема 1:
- существующий диалог из истории открывается и работает нормально
- но новый чат не стартует корректно
- настройки “запросить имя” и “запросить телефон” включены, но при первом диалоге они не срабатывают

Проблема 2:
- при переходе из заказа в клиента открывается не карточка клиента, а общий список клиентов

Нужно найти root cause и исправить это системно.

==================================================

Сейчас:

- история читается
- отправка в существующий диалог идет
- но новый чат не стартует как надо

Это значит, что broken flow именно в create new session / first-contact flow.

Проверь:

1. create new chat session logic
2. localStorage / external id logic
3. sessionToken generation
4. initial widget state
5. when no existing session is found
6. create-session API call
7. fallback if session create fails
8. pre-chat form logic
9. welcome / first-message state
10. how widget decides “new conversation” vs “existing conversation”

Нужно:
- при отсутствии существующей session создавать новую корректно
- новый чат должен действительно стартовать
- если session create failed, не делать вид, что все ок
- не должно быть сценария, где только старые диалоги работают, а новый не начинается

2. НАСТРОЙКИ “ЗАПРОСИТЬ ИМЯ / ТЕЛЕФОН” НЕ СРАБАТЫВАЮТ ПРИ ПЕРВОМ ДИАЛОГЕ
==================================================

- запросить имя
- запросить телефон

Но при первом диалоге виджет их не спрашивает.

Это критичный баг настройки → frontend flow.

Проверь весь pipeline:

1. settings save in CRM
2. DB values
3. public settings API
4. widget config fetch
5. pre-chat form rendering
6. required/optional logic
7. new session only logic
8. existing session bypass logic
9. whether settings are ignored after history is loaded
10. whether widget skips pre-chat form incorrectly

Нужно:
- если включено “запросить имя”, при первом новом диалоге должно появляться поле имени
- если включено “запросить телефон”, при первом новом диалоге должно появляться поле телефона
- если поля required, без них нельзя начать новый диалог
- если это повторное открытие уже существующей session, повторно спрашивать не нужно
- но именно на первом новом диалоге форма обязана срабатывать

Поля должны:
- реально отображаться
- реально валидироваться
- реально сохраняться в chat session / metadata / linked customer lead

==================================================
3. НУЖЕН ЧЕТКИЙ ПРИНЦИП:
НОВЫЙ ЧАТ = PRE-CHAT FLOW
СУЩЕСТВУЮЩИЙ ЧАТ = HISTORY FLOW
==================================================

Сделай логику явной.

Если:
- session не найдена
- пользователь открывает чат впервые
=> запускать pre-chat flow

Если:
- session уже есть
=> открывать историю диалога

Pre-chat flow должен учитывать настройки:
- collect_name_enabled
- collect_phone_enabled
- collect_email_enabled
- require_name
- require_phone
- require_email

После успешного pre-chat step:
- создается session
- сохраняется внешний id / sessionToken
- данные сохраняются
- открывается диалог
