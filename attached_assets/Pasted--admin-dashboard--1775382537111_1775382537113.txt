==================================================
2. ПОКУПКА / БРОНИРОВАНИЕ ТУРА ДОЛЖНЫ БЫТЬ ОРГАНИЗОВАНЫ НОРМАЛЬНО
==================================================

Сейчас нужен настоящий сценарий бронирования/покупки тура.

Пользовательский путь должен быть таким:
1. пользователь открывает список туров
2. открывает карточку тура
3. видит:
   - описание
   - программу
   - сложность
   - цену
   - депозит
   - включено / не включено
   - фотографии
   - ближайшие даты
   - доступные места
4. выбирает дату тура
5. выбирает количество участников
6. заполняет контакты
7. оформляет бронь / покупку
8. создается `tour_booking`
9. менеджер получает уведомление
10. бронирование видно в CRM

Нужно:
- чтобы это был реальный flow
- не просто кнопка “Оставить заявку” без логики
- а именно рабочая запись/покупка тура

  - planned

В CRM менеджер должен иметь возможность:

- менять депозит даты
- менять количество мест

- видеть, сколько мест уже занято
- закрывать запись на дату, если мест нет
