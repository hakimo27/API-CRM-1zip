==================================================
16. ИНВЕНТАРЬ ДОЛЖЕН ВЛИЯТЬ НА ДОСТУПНОЕ КОЛИЧЕСТВО ТОВАРА
==================================================

Сейчас есть критичная проблема: когда в инвентаре единицу товара списывают или переводят в обслуживание/ремонт, доступное количество товара для аренды не меняется. Это неправильно. Если единица выбыла из использования, доступность товара должна уменьшаться.

Нужно исправить это системно.

### Проблема
Сейчас при действиях в inventory:
- списание
- перевод в ремонт
- перевод в некомплект
- вывод из доступных
- другие статусы недоступности

товар по-прежнему может отображаться как доступный в прежнем количестве.

Это ломает:
- availability
- rental order create/edit
- checkout
- остатки в карточке товара
- календарь доступности

### Правильная логика
Если inventory unit переведен в статус:
- in_repair
- incomplete
- written_off
- unavailable / аналогичный недоступный статус

то эта единица больше не должна учитываться как доступная для аренды.

Если inventory unit переведен обратно в:
- available

то доступность должна увеличиваться обратно.

### Что нужно проверить
1. как сейчас считается totalStock / available quantity
2. учитывается ли inventory_units.status
3. есть ли пересчет после изменения статуса inventory unit
4. связан ли inventory с products aggregate quantity
5. есть ли broken sync между inventory и rental availability
6. не используется ли в каталоге устаревшее сохраненное totalStock без учета статусов inventory

### Что нужно сделать
1. Привязать inventory status к доступному количеству товара
2. Пересчитывать доступность после:
   - списания
   - перевода в ремонт
   - перевода в некомплект
   - возврата в available
3. Обновить:
   - availability service
   - карточку товара
   - список товаров аренды
   - create/edit rental order
   - checkout
   - availability calendar
4. Показать менеджеру агрегированные значения:
   - всего единиц
   - доступно
   - занято
   - в ремонте
   - некомплект
   - списано

### Важно
Нельзя, чтобы inventory жило отдельно, а rental stock считался сам по себе.
Inventory должен реально влиять на доступность товара.

### Что показать в ответе
Покажи:
1. Почему раньше статус inventory unit не влиял на количество
2. Как теперь inventory влияет на availability
3. Какие статусы уменьшают доступное количество
4. Где теперь пересчитываются остатки
5. Какие файлы изменены
6. Какие ручные тесты проведены
