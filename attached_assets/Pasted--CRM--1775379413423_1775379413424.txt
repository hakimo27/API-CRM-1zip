Исправь раздел “Страницы” в CRM. Сейчас в пункте “Страницы” показывается пусто, хотя страницы в системе есть, например “О нас”. Из-за этого невозможно нормально редактировать существующие страницы сайта.

Нужно не просто показать пустой список, а сделать полноценное управление страницами.

==================================================
1. ПРОБЛЕМА
==================================================

Сейчас:
- в меню есть раздел “Страницы”
- но внутри пусто
- при этом страницы сайта существуют, например:
  - О нас
  - Контакты
  - Доставка
  - FAQ / другие static pages
- менеджер не понимает, где и как их редактировать

Это значит, что broken один из сценариев:
- страницы не грузятся из БД
- список страниц не подключен к API
- используются hardcoded/static pages вне CRM
- pages table есть, но UI не подключен
- route есть, но loader broken
- фильтр active/published скрывает записи
- pages реально не импортированы/не созданы как сущности

==================================================
2. ЧТО НУЖНО ПРОВЕРИТЬ
==================================================

Проверь весь pipeline:

1. Есть ли таблица `pages`
2. Есть ли реальные записи в `pages`
3. Есть ли public pages, которые до сих пор захардкожены в коде
4. Есть ли admin API:
   - GET /api/admin/pages
   - GET /api/admin/pages/:id
   - POST /api/admin/pages
   - PATCH /api/admin/pages/:id
   - DELETE /api/admin/pages/:id
5. Есть ли CRM pages list page
6. Есть ли pages details/edit page
7. Есть ли broken mapping between slug/title/content and site routes
8. Не скрываются ли страницы из-за фильтра active
9. Не используется ли не та таблица/источник данных

4. СУЩЕСТВУЮЩИЕ СТРАНИЦЫ ДОЛЖНЫ ПОЯВИТЬСЯ В CRM
==================================================

Нужно, чтобы существующие страницы сайта тоже были доступны для редактирования.

Например:
- О нас
- Контакты
- Доставка

- FAQ / условия / политика и другие static pages

Если сейчас эти страницы захардкожены:

- либо связать текущие routes с таблицей `pages`

То есть:
- если страница уже существует на сайте
- она должна появиться в CRM Pages list
- и открываться на редактирование
