Исправь и улучши экран CRM Settings и публичную главную страницу. Сделай это как нормальную production-ready админку, а не набор технических полей.

====================
1. ПЕРЕРАБОТАТЬ UX СТРАНИЦЫ SETTINGS
====================

Сейчас на странице настроек возле каждого input есть отдельная кнопка сохранения. Это неудобно, перегружает интерфейс и выглядит как технический прототип.

Нужно полностью переработать страницу настроек.

Сделай логику такой:
- одна вкладка = одна форма
- можно изменить сразу несколько полей
- у формы одна общая кнопка сохранения
- при необходимости кнопка “Отменить”
- показывать состояние “Есть несохраненные изменения”
- показывать loading state при сохранении
- показывать toast/success после сохранения
- показывать toast/error при ошибке
- если изменений нет, кнопка “Сохранить” должна быть disabled

Пример:
- вкладка “Контакты” → редактируем все поля → одна кнопка “Сохранить контакты”
- вкладка “Брендинг” → одна кнопка “Сохранить брендинг”
- вкладка “Telegram” → одна кнопка “Сохранить настройки Telegram”
- вкладка “Бронирование” → одна кнопка “Сохранить настройки бронирования”

Ожидаемый результат:
- больше нет отдельной кнопки save возле каждого поля
- сохранение идет всей секцией / вкладкой
- UX выглядит как полноценная CRM

====================
3. СДЕЛАТЬ ОТДЕЛЬНЫЙ ПУНКТ НАСТРОЕК “ДОСТАВКА”
====================

Нужно добавить отдельную вкладку / раздел в CRM Settings:
- Доставка

Этот раздел должен управлять логикой доставки на сайте.

Поля:
- delivery.enabled — включена ли доставка вообще
- delivery.self_pickup_enabled — включен ли самовывоз
- delivery.text_short — короткий текст для сайта
- delivery.text_full — подробное описание условий доставки
- delivery.price_from — цена от
- delivery.free_from_amount — бесплатная доставка от суммы
- delivery.manager_required — доставка только после согласования менеджером
- delivery.regions — список доступных зон / городов / районов
- delivery.schedule — график доставки
- delivery.same_day_enabled — доставка день в день
- delivery.notes — примечания
- delivery.show_on_site — показывать ли блок доставки на сайте

Переключатели:
- Включить доставку
- Включить самовывоз
- Показывать блок доставки на сайте
- Требовать подтверждение менеджера для доставки
- Разрешить доставку день в день

Логика сайта:
- если delivery.enabled = false, на сайте не показывать доставку как доступную опцию
- если delivery.self_pickup_enabled = true, самовывоз остается доступным
- если и доставка, и самовывоз отключены, система должна показать предупреждение в CRM
- checkout должен использовать эти настройки
- публичные страницы “Доставка” и “Контакты” должны брать данные из settings

Ожидаемый результат:
- в CRM есть отдельная понятная вкладка “Доставка”
- можно включать/выключать доставку без разработчика
- сайт реально использует эти настройки
