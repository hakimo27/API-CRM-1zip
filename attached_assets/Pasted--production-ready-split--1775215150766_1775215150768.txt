Сделай production-ready split-проект для компании по аренде байдарок, продаже снаряжения и организации туров/сплавов.

ОБЯЗАТЕЛЬНЫЙ СТЕК:
- Frontend: React + Next.js + TypeScript
- Backend: Node.js + NestJS + TypeScript
- Database: PostgreSQL
- Cache / queues / realtime support: Redis
- ORM: Drizzle ORM
- Admin/CRM: отдельное React/Next.js admin-приложение
- API: REST first
- Auth: JWT + refresh tokens + email verification + password reset
- Email: SMTP abstraction
- File storage: local/S3-compatible abstraction
- Chat: WebSocket + polling fallback
- Telegram bot integration
- Monorepo

Сделай именно split architecture:
- apps/web — публичный сайт
- apps/admin — CRM / админка
- apps/api — NestJS backend
- packages/shared — общие типы, DTO, enums, utils
- packages/ui — общие UI-компоненты
- packages/config — shared configs

17. ПОЛНОЦЕННЫЙ ДВУСТОРОННИЙ ЧАТ САЙТ ↔ TELEGRAM
================================

ОБЯЗАТЕЛЬНО:
Нужна двусторонняя синхронизация переписки:

1. Пользователь пишет в чат-виджет на сайте
2. Сообщение сохраняется в БД
3. Сообщение автоматически уходит в Telegram-бота / Telegram-группу менеджеров
4. Менеджер отвечает в Telegram
5. Ответ автоматически возвращается в виджет сайта
6. Вся история сообщений сохраняется в CRM
7. Сообщения привязаны к:
   - chat_session
   - customer
   - order / rental_order / sale_order / tour_booking
   - domain/subdomain
   - assigned manager

Это должна быть настоящая двусторонняя система, а не просто уведомления.

Website chat widget:
- работает на всех доменах и субдоменах
- открывается в правом нижнем углу
- работает без перезагрузки
- поддерживает anonymous session
- после логина или заказа связывается с customer
- хранит историю сообщений
- прикрепляет order / sale order / tour booking
- поддерживает quick actions
- показывает read/unread
- показывает статус диалога
- имеет websocket + polling fallback
- подготовлен под attachments

Quick actions:
- Подобрать байдарку
- Проверить свободные даты
- Вопрос по аренде
- Купить байдарку

- Связаться с менеджером

Telegram integration:

- ответ из Telegram должен вернуться в конкретный сайт-чат
